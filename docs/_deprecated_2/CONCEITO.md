# METODOLOGIA DRY - PROTOCOLO DE IMPLEMENTAÇÃO

Seguir o fluxo abaixo para implementar funcionalidades baseadas em componentes DRY.

## FLUXO

1. Abrir `/docs/IMPLEMENTS.md`
2. Achar o `[*]`. Esta será a funcionalidade a ser implementada.
3. Ler descrição, componentes utilizados e comportamento esperado.
4. Montar um checklist de implementação:
   - 1 item para a funcionalidade
   - 1 item para cada dependência (componente DRY)
5. Para cada dependência:
   - Buscar no código
   - `[x]`: reutilizar
   - `[ ]`: implementar
6. Implementar tudo
7. Mudar status para `[x]`

---

## REGRA SIMPLES

**Tem itálico + crase?** → Definição, não busque
**Só colchetes?** → Referência, busque
