# Templates DRY

Objetivo: padronizar a documentação de novos componentes DRY com um modelo único.

## Como usar
1) Copie `ui-component.md` para o local desejado e preencha os campos.
2) Mantenha o ID no formato `*` + backticks, ex.: *`DRY.UI:SEU_COMPONENTE`*.
3) Use as seções opcionais (Dependências, Fluxo, Variantes) apenas se fizer sentido.

## Escopo
- Componentes base (`DRY.BASE-UI:*`) e especializados (`DRY.UI:*`) usam o mesmo template.
- Inclua referências cruzadas para reutilização e rastreabilidade (DRY.UI, DRY.CONCEPT, DRY.OBJECT).
